INTRODUCTION                                    *semantic-ctags-diff-intro*

vim-semantic-ctags-diff provides two independent features:

1. Semantic branch diffs — calls the external Python tool semantic-branch-diff
   (PyDriller + ctags) and opens Markdown/JSON scratch buffers.

2. Difftastic in Vim — a Fugitive-style file diff rendered with difftastic
   (difft) via :Gdifftastic and :Gvdifftastic. This ports difftastic's
   structural diff into a Vim scratch window.

The plugin is compatible with Vim 8. It does not require Neovim or Lua. The two
features are independent: difftastic needs only git + difft, and the Python
module gracefully handles difftastic being absent.

==============================================================================
REQUIREMENTS                                *semantic-ctags-diff-requirements*

Required for semantic diff:
  - Vim 8+
  - Git
  - Python 3 (PyDriller and python-ctags3 importable by that interpreter)
  - Universal Ctags (or Exuberant Ctags with reduced C++ metadata)

Required for :Gdifftastic / :Gvdifftastic:
  - Git
  - difftastic (difft) on PATH

The semantic-branch-diff package does not need to be pip-installed. The plugin
runs submodules/semantic-ctags-diff via PYTHONPATH and python -m semantic_branch_diff.cli.

Ctags does NOT need JSON output support. The Python tool writes a classic tags
file (-f tags) and parses it with python-ctags3. Do not pass --output-format=json
to ctags via g:semantic_ctags_diff_extra_args. :SemanticCtagsDiffJson produces a
JSON semantic diff report from the Python tool, not JSON from ctags.

Optional:
  - vim-fugitive (recommended for submodule worktree detection)
  - vim-flog (optional log navigation companion)

Python library location (default auto-detect):
  submodules/semantic-ctags-diff/pyproject.toml

Fallback typo-tolerant path:
  submodules/sematic-ctags-diff/

INSTALLATION                                    *semantic-ctags-diff-install*

1. Install the plugin with your plugin manager, e.g. vim-plug: >

    Plug 'rafaelrojasmiliani/ctags-difftastic-semantic-diff-vim'

2. Initialize the submodule: >

    git submodule update --init --recursive

3. Ensure Python can import PyDriller and python-ctags3 (system packages or your
   own virtualenv). The plugin does not require pip install of semantic-branch-diff.

4. Generate help tags: >

    :helptags PATH_TO_PLUGIN/doc

COMMANDS                                        *semantic-ctags-diff-commands*

:SemanticCtagsDiff [base] [head]                *:SemanticCtagsDiff*
      Run semantic branch diff and open Markdown in a scratch buffer.
      Defaults: base = g:semantic_ctags_diff_default_base (main),
                head = g:semantic_ctags_diff_default_head (HEAD).

      Examples: >
        :SemanticCtagsDiff
        :SemanticCtagsDiff main
        :SemanticCtagsDiff main feature/foo
        :SemanticCtagsDiff origin/main HEAD

:SemanticCtagsDiffJson [base] [head]            *:SemanticCtagsDiffJson*
      Same as above but opens pretty-printed JSON.

      Example: >
        :SemanticCtagsDiffJson origin/main HEAD

:SemanticCtagsDiffCurrent                       *:SemanticCtagsDiffCurrent*
      Shortcut using configured default base and head.

:SemanticCtagsDiffMain                            *:SemanticCtagsDiffMain*
      Shortcut for: >
        :SemanticCtagsDiff main HEAD

:SemanticCtagsDiffOriginMain                      *:SemanticCtagsDiffOriginMain*
      Shortcut for: >
        :SemanticCtagsDiff origin/main HEAD

:SemanticCtagsDiffRefresh                         *:SemanticCtagsDiffRefresh*
      Re-run the last semantic diff (bypasses the /tmp cache) and replace the
      scratch buffer when invoked from the previous result buffer.

:SemanticCtagsDiffClearCache                       *:SemanticCtagsDiffClearCache*
      Delete all cached result files from g:semantic_ctags_diff_cache_dir.

:SemanticCtagsDiffCopyCommand                     *:SemanticCtagsDiffCopyCommand*
      Echo and copy the last external shell command to the clipboard register.

:SemanticCtagsDiffDebugLog                        *:SemanticCtagsDiffDebugLog*
      Open a copyable debug log scratch buffer.

:SemanticCtagsDiffClearDebugLog                   *:SemanticCtagsDiffClearDebugLog*
      Clear the in-memory debug log.

:SemanticCtagsDiffFlog [base] [head]              *:SemanticCtagsDiffFlog*
      (vim-flog only) Open Flog for base..head as a raw git log companion.

:SemanticCtagsDiffFlogSymbol                      *:SemanticCtagsDiffFlogSymbol*
      (vim-flog only) Pick a modified symbol from the last JSON result and
      open its line-range history. By default this opens a NEW TAB (commit
      graph + diff), like plain :Flog. Set g:semantic_ctags_diff_flog_open to
      'Flogsplit' to open a split instead.

:FlogSymbol / :FlogFunction / :FlogClass / :FlogNamespace   *:FlogSymbol*
      (vim-flog only) History of the symbol under the cursor in a NEW TAB
      (commit graph + diff), like :Flog. Defined only if not already defined.

:FlogsplitSymbol / :FlogsplitFunction / ...        *:FlogsplitSymbol*
      (vim-flog only) Same, but in a split of the current window.

:Gdifftastic [ref]                                *:Gdifftastic*
      Difftastic diff of the current file against [ref] (default HEAD), in a
      horizontal scratch split. See |semantic-ctags-diff-difftastic|.

:Gvdifftastic [ref]                               *:Gvdifftastic*
      Same as :Gdifftastic, in a vertical split.

DIFFTASTIC                                      *semantic-ctags-diff-difftastic*

This plugin ports difftastic (a structural, syntax-aware diff tool) into Vim.
:Gdifftastic and :Gvdifftastic mirror Fugitive's file diff, but render the diff
with difft instead of Vim's line-based diff.

They run, for the current file: >

    DFT_DISPLAY=<display> DFT_COLOR=never \
      git -C <worktree> -c diff.external=difft --no-pager diff --ext-diff <ref> \
      -- <file>

difftastic receives the old/new blobs directly from git, so the result matches
what Fugitive would diff. Output opens in a read-only scratch buffer with a
header (file, ref, repo, exact command).

Requirements and notes:
  - difft must be on PATH (or set g:semantic_ctags_diff_difft).
  - Works with only git + difft; no Python or ctags needed.
  - Commands are defined only if not already present, so a custom :Gdifftastic
    is never overridden.
  - Output is plain text (no ANSI colors); this is not true vimdiff mode.

Examples: >

    :Gdifftastic
    :Gvdifftastic origin/main
    :Gdifftastic HEAD~3

CONFIGURATION                                   *semantic-ctags-diff-config*

g:semantic_ctags_diff_python                      *g:semantic_ctags_diff_python*
      Python executable. Default: 'python3'

g:semantic_ctags_diff_root                        *g:semantic_ctags_diff_root*
      Optional explicit path to the Python project. When empty, auto-detect
      submodules/semantic-ctags-diff relative to repo or file directory.

g:semantic_ctags_diff_cli                         *g:semantic_ctags_diff_cli*
      Optional explicit shell prefix overriding the default module invocation.
      When empty, the plugin runs: >
        PYTHONPATH=<submodule> python3 -m semantic_branch_diff.cli
      using g:semantic_ctags_diff_python, or submodules/semantic-ctags-diff/.venv/bin/python3
      when that venv exists (deps-only venv; package install not required).

g:semantic_ctags_diff_default_base                *g:semantic_ctags_diff_default_base*
      Default base ref. Default: 'main'

g:semantic_ctags_diff_default_head                *g:semantic_ctags_diff_default_head*
      Default head ref. Default: 'HEAD'

g:semantic_ctags_diff_ctags                       *g:semantic_ctags_diff_ctags*
      ctags executable. Default: 'ctags'

g:semantic_ctags_diff_format                      *g:semantic_ctags_diff_format*
      Default format (used by configuration only). Default: 'markdown'

g:semantic_ctags_diff_include                     *g:semantic_ctags_diff_include*
      Comma-separated file extensions. Default C/C++ set.

g:semantic_ctags_diff_open_cmd                    *g:semantic_ctags_diff_open_cmd*
      Window command for scratch buffers. Default: 'botright new'

g:semantic_ctags_diff_debug                       *g:semantic_ctags_diff_debug*
      Pass --debug to the Python CLI. Default: 0

g:semantic_ctags_diff_use_fugitive_worktree       *g:semantic_ctags_diff_use_fugitive_worktree*
      Use FugitiveWorkTree() when vim-fugitive is available. Default: 1

g:semantic_ctags_diff_extra_args                  *g:semantic_ctags_diff_extra_args*
      List of extra CLI arguments appended to the command.

g:semantic_ctags_diff_cache                       *g:semantic_ctags_diff_cache*
      Cache successful diff results under /tmp. Default: 1

g:semantic_ctags_diff_cache_dir                   *g:semantic_ctags_diff_cache_dir*
      Directory for cached .markdown and .json files. Default:
      '/tmp/semantic_ctags_diff'. Nothing is written into the Git work tree.

g:semantic_ctags_diff_flog_open                   *g:semantic_ctags_diff_flog_open*
      Flog command used by :SemanticCtagsDiffFlogSymbol. 'Flog' opens a new
      tab (commit graph + diff), 'Flogsplit' opens a split. Default: 'Flog'

g:semantic_ctags_diff_difft                       *g:semantic_ctags_diff_difft*
      difftastic executable for :Gdifftastic. Default: 'difft'

g:semantic_ctags_diff_difftastic_display          *g:semantic_ctags_diff_difftastic_display*
      DFT_DISPLAY value: 'side-by-side' or 'inline'. Default: 'side-by-side'

g:semantic_ctags_diff_difftastic_context          *g:semantic_ctags_diff_difftastic_context*
      DFT_CONTEXT lines around each change. Default: 3

Example ~/.vimrc: >

    let g:semantic_ctags_diff_default_base = 'develop'
    let g:semantic_ctags_diff_debug = 1
    let g:semantic_ctags_diff_extra_args = ['--no-pydriller-methods']
    let g:semantic_ctags_diff_difftastic_display = 'inline'

SUBMODULES                                  *semantic-ctags-diff-submodules*

Repository root is resolved via:
  1. FugitiveWorkTree() when vim-fugitive is loaded
  2. git -C <file-dir-or-cwd> rev-parse --show-toplevel

Python sources are resolved separately (no pip install required):
  1. g:semantic_ctags_diff_root if set
  2. submodules/semantic-ctags-diff next to the plugin directory
  3. submodules/semantic-ctags-diff under the Git repo root
  4. Walk upward from the current file directory

Invocation (default): >
  PYTHONPATH=<submodule> python3 -m semantic_branch_diff.cli --repo ... --base ... --head ...

The --repo argument passed to semantic-branch-diff is always the detected Git
worktree root.

==============================================================================
FUGITIVE AND FLOG                           *semantic-ctags-diff-integration*

vim-fugitive
  Improves repo root detection in submodule worktrees. The plugin does not
  require fugitive but enables g:semantic_ctags_diff_use_fugitive_worktree by
  default.

vim-flog
  Optional companion commands:
    :SemanticCtagsDiffFlog
    :SemanticCtagsDiffFlogSymbol

  Branch-diff JSON includes a top-level "navigation" list (built by Python) with
  label and flog_limit for each modified symbol.

  Cursor symbol history (:FlogsplitSymbol, :FlogsplitFunction, etc.) is provided
  by plugin/semantic_ctags_flog.vim when not already defined. These use Python
  symbol-at mode — not Vim ctags parsing.

  semantic-branch-diff --symbol-at --file PATH --line N [--kind function]

PYTHON VS VIM                               *semantic-ctags-diff-architecture*

Python (semantic-branch-diff):
  - ctags via classic tags file (not ctags JSON output)
  - Symbol normalization, branch diff, navigation / flog_limit helpers

Vim plugin:
  - Editor integration, scratch buffers, Fugitive worktree, Flog UI
  - Difftastic file diff (:Gdifftastic) via git external diff
  - Does not re-implement ctags parsing from legacy files/ scripts

Not included in this plugin:
  - Difftastic staged-diff commit message context
  - Fugitive :Commit ctags comment injection (see files/ reference script)

MAPPINGS                                      *semantic-ctags-diff-mappings*

Report buffer maps ~
  Inside the :SemanticCtagsDiff Markdown report, buffer-local maps jump from a
  symbol line to its source (mirroring vim-fugitive):

    <CR>   open the symbol in the current window
    o      open in a horizontal split
    gO     open in a vertical split
    O      open in a new tab

  Added / modified symbols open the working-tree file at the symbol line,
  falling back to :Gedit <head>:<path> when the file is not in the current
  checkout. Removed symbols open :Gedit <base>:<path> (the base commit where
  the symbol still exists). Added symbols use the cached JSON for their path,
  and the fugitive fallback requires vim-fugitive.

No global mappings are defined by default. Suggested mappings: >

    nnoremap <leader>sd :SemanticCtagsDiff<CR>
    nnoremap <leader>sj :SemanticCtagsDiffJson<CR>
    nnoremap <leader>sr :SemanticCtagsDiffRefresh<CR>
    nnoremap <leader>sl :SemanticCtagsDiffDebugLog<CR>
    nnoremap <leader>dt :Gvdifftastic<CR>

TROUBLESHOOTING                        *semantic-ctags-diff-troubleshooting*

ctags not found ~
  Install Universal Ctags and ensure it is on PATH, or set: >
    let g:semantic_ctags_diff_ctags = '/usr/bin/ctags'

ctags JSON output not required ~
  This plugin never uses ctags --output-format=json. A normal Universal or
  Exuberant ctags build is sufficient. The semantic-branch-diff tool invokes
  ctags with -f and reads the classic tags file via python-ctags3.

difftastic (difft) not found ~
  :Gdifftastic and :Gvdifftastic need difftastic on PATH. Install it (e.g.
  brew install difftastic, cargo install difftastic) or set: >
    let g:semantic_ctags_diff_difft = '/path/to/difft'

:Gdifftastic shows no diff ~
  The current buffer must be a saved file inside the Git worktree, and it must
  differ from [ref] (default HEAD). Save the buffer first.

Python import errors (PyDriller / python-ctags3) ~
  The submodule is run from source; only runtime libraries must be importable.
  Use a Python that already has them, or a deps-only venv, e.g.: >
    python3 -m venv submodules/semantic-ctags-diff/.venv
    submodules/semantic-ctags-diff/.venv/bin/pip install pydriller python-ctags3
  Or set g:semantic_ctags_diff_python to your interpreter path.

submodules/semantic-ctags-diff not found ~
  Run: >
    git submodule update --init --recursive
  Or set g:semantic_ctags_diff_root to the semantic-ctags-diff source directory.

Wrong repo root inside submodule ~
  Install vim-fugitive or verify: >
    git -C . rev-parse --show-toplevel
  Ensure g:semantic_ctags_diff_use_fugitive_worktree = 1.

No changed files ~
  The report may be empty when base and head have no analyzed file changes.
  Widen --include extensions via g:semantic_ctags_diff_include.

Command too slow ~
  Semantic diffs run synchronously. Large branches with many C++ files can take
  minutes. Use :SemanticCtagsDiffDebugLog to inspect timing and command output.
  Consider g:semantic_ctags_diff_extra_args = ['--no-pydriller-methods'].

General debugging ~
  Enable debug and inspect the log: >
    let g:semantic_ctags_diff_debug = 1
    :SemanticCtagsDiff main HEAD
    :SemanticCtagsDiffDebugLog
